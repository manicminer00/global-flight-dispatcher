User Preferences

⭐ Owned Airports Filter
Enter the ICAO codes for airport scenery that you own, separated by commas. As long as the ICAO is in the master database, or you have added it to your custom database below, and you have selected 'Enable Preferred Routing', the dispatcher will prioritize routes using these airports and highlight the ICAO code in a green colour on the dispatch form to indicate that you own this airport.
📍 Add New Airport (to Local DB)
Adds a missing airport to your local database so it can be used for departures, arrivals, and owned-airport routing. Latitude and longitude are required for distance and route generation.
Why separate turboprop and business jet?
Even when a runway is long enough on paper, noise rules, weight limits, or missing FBO facilities may still restrict it to turboprops only. Set the highest aircraft category this airport actually supports. Runway length is entered separately and is also checked against each aircraft.
🛩️ Add New Aircraft (to Local DB)
Adds a missing aircraft to your local fleet list.
Minimum sector distance is set automatically from the aircraft class. Enter manufacturer still-air maximum range from the POH or AFM — the dispatcher uses this for long-haul routing and range-cap alerts.

Additional Options

What does Add New Airport (to Local DB) or Add New Aircraft (to Local DB) do?
📍 Add New Airport (to Local DB):
  • Saves a missing airport to your browser so it appears in the departure search and route picker.
  • Runway length and largest permitted aircraft type control which airframes can use the airport.
  • Coordinates are used for distance, heading, and map routing — look them up if you are unsure.
🛩️ Add New Aircraft (to Local DB):
  • Saves a missing aircraft to your local fleet dropdown.
  • Mission tags are used to define how the dispatcher classifies aircraft. Gliders need class Glider / Sailplane, category Glider, and the appropriate glider options (self-launch, sailplane, etc.).
  • Enter maximum range (nm) from the aircraft manual — this controls short-haul and long-haul routing. Minimum sector distance is set automatically by class.
  • Military airlifters (e.g. A400): class Turboprop, tick Military Aircraft, category Heavy Cargo / Airlifter, primary use Cargo / freight.
  • Custom glider strips: set largest permitted aircraft type to Glider strip / winch launch only.

What does Reset Custom Database do?
Step-by-Step
  • 1. Confirmation: A dialog asks for permission to wipe custom data.
  • 2. Deletion: Only the (dispatcher_custom_airports, and dispatcher_custom_fleet) keys are removed from your browser's local memory.
  • 3. Reassurance: This does not clear cookies, bookmarks, or browsing history; it only affects this specific dispatcher tool.
  • 4. What about my Logbook?: 'Reset Custom Database' only deletes custom airframes and custom airports that you have manually added. It does not clear your Logbook or remove your Owned Airport list.
  • 5. Clean up: The interface instantly updates to reflect the empty database.

SimBrief Airport and Aircraft Profiles Notice
NOTAM: Several Asobo airports use custom identifier codes instead of the real-world code. VECTOR sends the real-world identifier to SimBrief to avoid routing errors and the custom Asobo identifier code will be shown in the Job Ticket, so that you can find the airport in the MSFS 2024 world map.

Some airports and aircraft in VECTOR's database are not included in the SimBrief database. To generate accurate flight plans in SimBrief, aircraft require an aircraft profile. If an airport or an aircraft is not included in the SimBrief database, and you cannot add it as a custom aircraft, you can use LittleNavMap to plan and follow your flight instead.
